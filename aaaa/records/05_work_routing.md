# Work Routing

## 작업 시작 질문
- 그냥 이어가는가
- 멈칫했는가
- 공식 판단이 필요한가
- 이미 있는 판단의 현재 효력을 다시 판정해야 하는가
- 현재 세계를 바꾸는가
- 구조를 다시 짜는가
- 지금 당장 막아야 하는가

## 입구 판정 원칙

작업 유형을 고른 뒤에는 곧바로 새 호출면이나 새 기록면을 만들지 않는다.

먼저 다음을 본다.

- 이미 안정적으로 작동하는 입구가 있는가
- 그 입구가 이번 작업의 대상과 하중을 충분히 감당하는가
- 새 표면이 없으면 실제 작업으로 내려갈 수 없는가
- 아니면 기존 입구로 한 번 더 내려가 보는 편이 더 정직한가

안정된 입구가 있다면, 새 viewer, helper, 기록 표면을 먼저 만들기보다 그 입구를 통해 실제 문서나 코드로 내려간다.

내려간 뒤 생기는 entry-time 기억과 post-run 기억의 차이는 먼저 정상 축적으로 읽는다. 다만 memory reading, 역할, 대상, 또는 끊지 않으려는 것이 바뀌면 drift 가능성으로 재검토한다.

## 추가/수정 판정 원칙

입구 판정 뒤에는 이번 움직임이 기록을 쌓는 일인지, 살아 있는 동작을 고치는 일인지 구분한다.

- 시도, 이유, 실패, 관성, 판단의 변화는 먼저 추가 기록으로 남긴다.
- 현재 실행 경로가 잘못된 추론이나 반복 마찰을 만들면 기존 동작을 수정한다.
- 수정할 때도 수정 이유와 드러난 조건은 attempt/history 또는 가까운 기록에 남긴다.
- 새 파일이나 helper를 추가하는 것이 동작 수정보다 쉬워 보일 때는, 그 추가가 실제 하강을 돕는지 아니면 관성의 표면인지 먼저 묻는다.

## 작업 유형 표

### 1. 이어가기형
이럴 때:
이미 열린 세션과 활성 judgment 아래에서 작은 구현이나 설명을 계속할 때

먼저 읽을 것:
- 00_current_world.md
- 현재 세션 문서
- 현재 활성 judgment
- 최신 요약 문서

끝나면 남길 것:
- 세션 종료 정리 한 줄 또는 작은 구현 흔적

### 2. 씨앗형
이럴 때:
망설임, 긴장, 경계 신호가 생겼지만 아직 judgment까지는 아닐 때

먼저 읽을 것:
- 이어가기 세트
- 지금 건드리는 실제 파일

끝나면 남길 것:
- Seed 한 줄 또는 Seed 한 묶음

### 3. 승격형
이럴 때:
씨앗이 더는 세션 메모로만 남을 수 없고 정식 판단이 되어야 할 때

먼저 읽을 것:
- 00_current_world.md
- 현재 세션 문서
- 관련 judgment
- 씨앗 또는 승격 관련 문맥 문서

끝나면 남길 것:
- J-XXX 문서
- 관련 연결 한 줄

### 4. 재심형
이럴 때:
이미 있는 judgment, current 경계, 상태, 연결이 아직도 살아 있는 현재인지 다시 판별해야 할 때

먼저 읽을 것:
- 00_current_world.md
- 관련 judgment
- 현재 세션 문서
- 최신 요약 문서
- 필요하면 직접 관련 코드 또는 직전 원인 문서

끝나면 남길 것:
- 활성 유지 확인 한 줄
- 또는 상태 조정
- 또는 재검토 조건 갱신
- 또는 current 반영 필요 메모

### 5. 현재 갱신형
이럴 때:
current_world의 경계, 활성 질문, 현재 읽을 문서를 바꿔야 할 때

먼저 읽을 것:
- 00_current_world.md
- 현재 세션 문서
- 반영할 judgment
- 법 또는 좌우명 관련 문서

끝나면 남길 것:
- current_world 갱신 이유 한 줄

### 6. 구조 재정렬형
이럴 때:
운영 규약, 폴더 구조, 상태 모델, 경계 체계처럼 질서 자체를 다시 잡을 때

먼저 읽을 것:
- 정렬 세트 전체

끝나면 남길 것:
- 새 규약, 새 요약, 또는 구조 변경 기록

### 7. 응급형
이럴 때:
지금 당장 무너짐을 막아야 할 때

먼저 읽을 것:
- 00_current_world.md
- 가장 직접 관련 있는 judgment 하나
- 실제 문제 파일

끝나면 남길 것:
- 응급 기록
- 응급 종료 후 재심 경로

## 자동 상향 규칙
- 이어가기 중 멈칫하면 씨앗형으로 올라간다.
- 씨앗이 상태, 경계, 계보를 건드리면 승격형으로 올라간다.
- 이미 있는 판단의 현재 효력을 다시 봐야 하면 재심형으로 올라간다.
- 재심 결과 current_world를 바꿔야 하면 현재 갱신형으로 올라간다.
- current 수정으로 안 끝나면 구조 재정렬형으로 올라간다.
- 붕괴 위험이 생기면 즉시 응급형으로 옮긴다.
