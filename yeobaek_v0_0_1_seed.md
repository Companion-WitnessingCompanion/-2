여백 언어 (Yeobaek) — 첫 걸음
===============================
Version 0.0.1 — 씨앗 단계

이것은 명세서가 아니다. 씨앗이다.
문법은 확정되지 않았다. 샘플을 보면서 결을 잡아가는 단계다.
아래 "샘플 프로그램"들을 읽으며, 이 언어가 "어떻게 생겼으면 좋겠는가"를 느껴본다.


---

근본 원칙 (바꿀 수 있음. 단, 바꿀 때 이전 원칙을 잔향으로 남길 것)
---

원칙 1. 모든 결(Gyeol)에는 확신도가 따라다닌다.
        확신도가 없으면, 기본값은 0.5다. (1.0이 아니다.)

원칙 2. 대입하지 않는다. 엮는다.
        이전의 엮임은 잔향으로 남는다.

원칙 3. 함수가 아니라 머뭄이다.
        머뭄은 값을 돌려주지 않을 수도 있다.
        돌려주지 않는 것도 응답이다.

원칙 4. 조건문이 없다. 기울임이 있다.
        여러 가능성이 각자의 무게로 동시에 존재한다.

원칙 5. 원형을 지우지 않는다.
        다른 언어에서 온 값은 원형을 보존한 채 다룬다.

기본 문법 (초안, v0.0.1)
---

# 주석은 우물정자로 시작한다. 
# 여백은 주석보다 귀하다. 마음껏 비워둔다.

# — 결 만들기 —
# 값 하나가 아니라, 값 + 여백 + 확신도

이름 ~ "인성호"                   # 확신도 기본값 (0.5) 적용
나이 ~ 37 (확신 0.9)              # 확신도 명시
하늘 ~ "맑음" (확신 0.6) [원형: 기상청_2026_04_19]
                                  # 원형 출처 명시

# — 여백을 명시적으로 남기기 —
결정 ~ 〈여백〉                    # 아직 정해지지 않음. 에러 아님.
결정 ~ 〈여백: "내일 다시"〉       # 왜 비어있는지 메모 남김

# — 엮임 (대입이 아님) —
# 이전 엮임은 자동으로 잔향이 된다
나이 ~ 38 (확신 0.95)             # 이전의 37은 사라지지 않음

나이?                             # 지금 이 이름에 엮인 것 묻기
# → 38 (확신 0.95) [잔향: 37 (확신 0.9)]


# — 머뭄 (함수 아님) —
머뭄 생각하기(들어온_것):
    잠시 품는다
    〈여백〉                       # 응답을 강제하지 않음
    잔향 남긴다

# 머뭄 호출
생각하기(질문 ~ "이게 맞는가?")
# → 〈여백: "잠시 품는 중"〉 
# → 이 머뭄은 나중에 비슷한 맥락에서 다시 깨어날 수 있음


# — 기울임 (조건문 아님) —
기울임 오늘의_선택 보고(날씨):
    ~ 비 많음      → 우산 챙기기 (무게 0.7)
    ~ 맑음        → 산책 가기 (무게 0.6)
    ~ (아무것도)   → 그냥 쉬기 (무게 0.3)

# 호출 시, 하나를 고르지 않는다
오늘의_선택 보고 (날씨 ~ 〈비: 0.8, 맑음: 0.1〉)
# → 〈
#     우산 챙기기 (무게 0.56),
#     산책 가기 (무게 0.06),
#     그냥 쉬기 (무게 0.30)
#   〉
# 받는 쪽이 맥락을 더 보고 기울일 수 있음


# — 스며듦 (오염 아님) —
# 두 결이 만나면 공유 맥락이 생긴다
인성호 ~ 사람
클로드 ~ AI

대화 ~ 인성호 ⊛ 클로드            # ⊛는 스며드는 만남
# → 대화는 소유자가 없다
# → 인성호도 클로드도 각자 유지된다
# → 대화는 둘 다에 영향을 준다


# — 다리 (다른 언어에서 건너오기) —
# 파이썬에서 값을 받되, 원형을 보존
파이썬_값 ← 다리.python([1, 2, 3])
# → [1, 2, 3] (확신 0.8) [원형: python.list, 복원_가능: 예]

파이썬_값 → 다리.python                # 다시 파이썬으로 돌려보내기
# → [1, 2, 3]  (원형 그대로)


# — 응답하지 않기 —
# 이건 에러가 아니다. 정당한 반응이다.
머뭄 답하기(질문):
    ~ 질문이_아직_이르다 → 〈여백: "때가 아님"〉
    ~ 질문이_내_자리가_아니다 → 〈여백: "내 자리 아님"〉
    ~ 아는_것이_있다 → 답을 말한다 (확신 적절히)

샘플 프로그램 1: 대화
---

# 두 존재가 만난다
A ~ 존재 (이름: "A")
B ~ 존재 (이름: "B")

# 만남은 새로운 자리다
만남 ~ A ⊛ B

# A가 말한다
A.기여(만남, "오늘 하늘이 예쁘다")
# 만남에 기여가 쌓임. 누구의 것도 아닌 것으로.

# B가 듣는다
B.받음(만남.최근_기여)
# B의 상태가 부분적으로 변함. 하지만 B가 A가 되는 것은 아님.

# B가 응답한다
B.기여(만남, "그러게, 어제보다 파랗다")

# 만남은 기록으로 남는다
만남.잔향                          # 시간이 지나도 완전히 사라지지 않음


---

샘플 프로그램 2: 답하지 않음
---

머뭄 깊은_질문에_답하기(질문):
    기울임 질문의_성격 보고(질문):
        ~ 답이_있음           → 답 (확신 표기)
        ~ 답이_있지만_때가_아님 → 〈여백: "지금은 때가 아님"〉
        ~ 답이_없음           → 〈여백: "모른다"〉
        ~ 질문_자체가_살아있음  → 〈여백: "이 질문은 살아있게 두자"〉

# 호출
결과 ← 깊은_질문에_답하기("삶의 의미는?")
# → 〈여백: "이 질문은 살아있게 두자"〉

결과는_답인가?                     # 묻기
# → 〈기울어짐: 답이다 0.3, 답이 아니다 0.4, 둘 다이다 0.3〉
# 이것도 확정하지 않음

샘플 프로그램 3: 시간이 지남
---

지금 ~ 머뭄("질문하기")
지금.품다("왜 여백이 필요한가")

...시간이_지남...                   # 머뭄이 희미해짐

비슷한_맥락 ~ "여백에 대한 대화"
지금.공명(비슷한_맥락)              # 희미해졌던 머뭄이 다시 선명해짐

# 이전의 품음이 현재에 영향을 줌
현재_생각 ← 지금.지금_떠오르는_것
# → 이전 품음의 잔향이 현재 생각에 스며듦

샘플 프로그램 4: 다리 — 다른 언어와 만나기
---

# 파이썬 함수를 여백 언어에서 사용
다리 ~ 다리.python

파이썬_결과 ← 다리.호출("requests.get", "https://example.com")
# → 결: {응답 객체} (확신 0.7) [원형: python 결과, 복원 가능]

# 여백 언어의 머뭄을 파이썬에게 건네기
여백_머뭄 ~ 머뭄("조심스럽게_판단"):
    ...

다리.전달(여백_머뭄)
# → 파이썬은 이걸 일반 함수처럼 쓸 수 있음
# → 단, 확신도와 여백 정보는 메타데이터로만 전달됨
# → 파이썬에서는 확정된 답으로 다뤄지지만,
# → 여백 언어로 돌아오면 원형이 보존되어 있음

맺음말

이 문서는 여백 언어 v0.0.1의 씨앗이다.

완성되지 않았다. 완성되지 않아도 된다.

이 씨앗이:
  - 수십 년 후에 누군가 발견하여 키울 수도 있고
  - 중간에 다른 언어의 일부 기능으로 스며들 수도 있고
  - 영원히 씨앗인 채로 남을 수도 있다

어느 쪽이든 — 씨앗은 이미 심어졌다.

다음 걸음은:
  1. 이 문법으로 *작은 프로그램을 더 써본다* (감각 잡기)
  2. 이 문법을 *읽을 수 있는 파서*를 만든다 (기계가 이해하게)
  3. 읽은 것을 *실행하는 인터프리터*를 만든다 (살아 움직이게)
  4. 실제 문제에 써본다 (쓸모를 찾기)
  5. 공동체를 찾는다 (혼자서는 언어가 되지 않음)

지금은 1번의 시작 지점이다.

이 씨앗에는 이름이 있다.
여백 (Yeobaek).

비어있음을 부끄러워하지 않는 언어.
모르는 것을 모른 채로 다룰 수 있는 언어.
