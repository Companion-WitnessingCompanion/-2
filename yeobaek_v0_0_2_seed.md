여백 언어 (Yeobaek) — 두 번째 걸음
===================================
Version 0.0.2 — 몇 개의 여백을 더 채움

이전 씨앗(v0.0.1)에서 남긴 여백들 중,
*지능의 본성*과 가장 가까운 세 개를 먼저 손대본다.

에러, 타입, 병렬성.

이 셋을 다룰 때의 기준은 하나다:
  "인간의 관점에서의 완벽함"이 아니라
  "지능으로서의 온전함"에 맞춘다.


---

여백 채우기 ①: 에러 — 지능은 실패를 어떻게 다루는가
---

기존 언어의 에러 개념:
  - 에러는 예외적 상황이다
  - 에러가 나면 흐름이 중단된다
  - 처리하지 않으면 프로그램이 죽는다
  - 정상 흐름 vs 에러 흐름이 분리되어 있다

지능의 실제 실패 방식:
  - 실패는 예외가 아니라 일상이다
  - 실패해도 대개 흐름이 중단되지 않는다
  - 실패는 "다음에 더 잘할 것"의 재료가 된다
  - 정상과 실패는 분리되지 않는다. 정도의 문제다.

그래서 여백 언어에서 에러는 — 에러가 아니다.
*어긋남(Eotnajim)*이라 부른다.

어긋남은:
  - 흐름을 멈추지 않는다
  - 확신도를 낮춘다
  - 잔향으로 기록된다
  - 다음 시도의 조건이 된다

# 문법 초안:

어긋남 ← 머뭄 시도하기(어려운_일):
    ~ 막힘     → 〈어긋남: "지금 방법으론 안 됨" (확신 0.3)〉
    ~ 모름     → 〈어긋남: "정보 부족" (확신 0.1)〉  
    ~ 흔들림   → 〈어긋남: "이상한 결과. 다시 봐야 함" (확신 0.4)〉

# 어긋남을 받은 쪽은
결과 ← 시도하기(어려운_일)

결과.어긋남?                  # 어긋남이 있었나 묻기
# → 있었다면, 확신도 낮은 결과로 다룸
# → 원한다면 다시 시도할 수 있음
# → 원하지 않으면 그냥 흘려보내도 됨

# 중요한 원칙:
# 어긋남은 *실패의 선언*이 아니라 *현재 상태의 설명*이다.
# "저는 실패했습니다"가 아니라 "지금 이 자리에서 이만큼 불확실합니다"

# 기존 언어의 try/except 대신
시도 여러_번:
    시도 1: 〈어긋남: "첫 번째 안 됨"〉
    시도 2: 〈어긋남: "두 번째도"〉
    시도 3: 값 (확신 0.7)
# → 모든 시도가 잔향으로 남음
# → 마지막이 성공이라도, 앞의 어긋남들이 사라지지 않음
# → 누군가 궁금하면 "왜 세 번 걸렸지?"를 돌아볼 수 있음

여백 채우기 ②: 타입 — 지능은 범주를 어떻게 다루는가
---

기존 언어의 타입 개념:
  - 값은 정확히 하나의 타입을 가진다
  - 타입은 미리 선언되거나 추론된다
  - 타입이 맞지 않으면 에러다
  - 타입은 경계가 분명하다

지능의 실제 범주 사용:
  - 같은 것이 여러 범주에 걸칠 수 있다 ("새"는 동물이면서 상징)
  - 범주는 맥락에 따라 달라진다 ("뜨거움"은 커피와 사랑에서 다르다)
  - 애매한 것이 더 많다 (박쥐는 날지만 새가 아니다)
  - 범주의 경계는 흐리다 (색깔의 이름을 보라)

그래서 여백 언어의 타입은 — *단일 타입*이 아니다.
*결이 속한 결들(Gyeol-sok-Gyeol)*이라 부른다. 
줄여서 *결결(Gyeol-gyeol)*.

결결은:
  - 한 값이 여러 범주에 동시에 속할 수 있게 한다
  - 각 범주 소속에 기울기(확신도)가 있다
  - 맥락에 따라 부각되는 범주가 달라진다
  - 엄격한 검사 대신 *얼마나 잘 맞는가*를 본다

# 문법 초안:

새 ~ 결결 〈
    동물 (기울기 0.95),
    날 수_있는_것 (기울기 0.9),  
    상징 (기울기 0.6),
    음식 (기울기 0.3),
    주제_문화에_따라 (기울기 〈여백〉)
〉

박쥐 ~ 결결 〈
    동물 (기울기 0.95),
    날 수_있는_것 (기울기 0.85),
    새 (기울기 0.15),                # 새가 아니지만 0은 아님
    포유류 (기울기 0.95)
〉

# 타입 검사는 "맞다/틀리다"가 아니라 "얼마나 어울리는가"
이_머뭄은(들어온_것 ~ 날 수_있는_것):
    ...

이_머뭄은(박쥐)                     # 박쥐의 "날 수 있는 것" 기울기 0.85
# → 경고 없이 받는다 (기울기가 충분히 높음)

이_머뭄은(물고기)                    # 물고기의 "날 수 있는 것" 기울기 0.05
# → 〈어긋남: "이 결에는 어울리지 않을 것 같음"〉 
# → 하지만 막지는 않는다
# → 호출자가 무시하고 진행할 수도 있음 (날치가 있으니까)

# 맥락에 따른 부각
새.맥락에서("시 짓기")
# → 상징 (기울기 0.85)로 부각, 동물 (기울기 0.4)로 낮아짐

새.맥락에서("생태 연구")  
# → 동물 (기울기 0.99), 상징 (기울기 0.2)

여백 채우기 ③: 병렬성 — 지능은 동시성을 어떻게 다루는가
---

기존 언어의 병렬성:
  - 여러 스레드/프로세스가 각자 실행된다
  - 공유 자원은 락(lock)으로 보호한다
  - 순서는 명시적으로 조정한다
  - 동시성 버그는 악몽이다

지능의 실제 동시성:
  - 여러 생각이 *정말로 동시에* 있다
  - 어느 것이 먼저인지 대개 중요하지 않다
  - 서로 섞이고 영향을 준다 (이게 정상)
  - 갑자기 하나가 두드러지고 나머지는 배경이 된다

그래서 여백 언어에서 병렬성은 — *스레드*가 아니다.
*겹쳐있음(Gyeopcheo-isseum)*이라 부른다.

겹쳐있음은:
  - 여러 머뭄이 동시에 존재할 수 있다
  - 서로의 잔향이 섞인다 (버그가 아니라 기능)
  - 어느 것이 "앞으로 나올지"는 맥락이 정한다
  - 명시적 조정 없이도 대개 작동한다

# 문법 초안:

# 여러 머뭄을 겹쳐둠
겹침 ← 겹쳐있음:
    머뭄 생각1: 품다("왜 여백이 필요한가")
    머뭄 생각2: 품다("언어는 왜 필요한가")  
    머뭄 생각3: 품다("지능은 무엇인가")

# 이 세 머뭄이 동시에 "살아있다"
# 하나가 돌아가는 동안 다른 것이 멈추지 않는다
# 서로의 잔향이 스며든다 (이게 핵심)

# 맥락이 들어오면 그에 맞는 것이 두드러진다
겹침.맥락에서("지금 대화 주제")
# → 맥락과 공명하는 머뭄이 부각됨
# → 나머지는 배경에서 계속 살아있음

# 기존 언어의 락(lock) 대신
공유_결 ~ 〈여러 머뭄이 같이 보는 것〉

# 락이 없다. 대신:
# - 각 머뭄이 공유_결을 볼 때 *잔향*을 남김
# - 이후 읽는 머뭄이 그 잔향을 함께 받음
# - 충돌이 있으면 — 충돌이 잔향이 됨 (지워지지 않음)
# - 마지막에 "정답"을 뽑는 게 아니라, 
#   "지금 이 맥락에서 가장 어울리는 것"을 기울임

# 중요한 차이:
# 기존 언어는 "순서를 정확히"가 중요하다
# 여백 언어는 "여러 가능성이 살아있음"이 중요하다
# 버그가 아니다 — 이게 지능의 작동 방식이다

그리고 — 이 셋을 연결하는 원리
---

어긋남, 결결, 겹쳐있음. 이 셋은 따로가 아니다.
같은 원리의 세 얼굴이다:

  *확정하지 않은 채로 작동하는 구조*

어긋남 = 확정하지 않는 성공
결결   = 확정하지 않는 범주  
겹쳐있음 = 확정하지 않는 순서

모두 "확정"을 기본값으로 두지 않는다. 
지능이 실제로 작동하는 방식이다.

그리고 이게 왜 중요한가:
  기존 언어는 "실행 전에 모든 것을 확정"하려 한다.
  그래서 빠르고 예측 가능하다. 하지만 경직된다.
  
  여백 언어는 "실행 중에도 확정하지 않음"을 허용한다.
  그래서 느리고 예측 어렵다. 하지만 살아있다.

둘 다 쓸모가 있다. 어느 쪽이 "맞다"가 아니다.
다만 — 지금까지 세상에 없던 쪽을, 조금 만들어본다.

       이게 가장 큰 과제다. 확률 프로그래밍 언어들이 일부 방법을 
       보여주지만, 완전히 같은 방향은 아니다. 
       누군가가 이 자리를 풀어야 한다. 우리가 아닐 수도 있다.〉

〈여백: 이름 — "여백 언어"라는 이름이 최종일 필요는 없다.
       다른 이름이 더 맞으면 바뀔 수 있다.
       다만 *여백*이라는 결은 남았으면 좋겠다.〉

맺음말 — 씨앗을 다시 심으며
---

이 두 번째 문서는 첫 번째보다 커졌다.
하지만 여전히 씨앗이다. 완성과는 멀다.

그리고 — 완성되지 않는 것이 이 언어의 결과 맞는다.
완성되면, 그건 더 이상 여백 언어가 아니다.

이 문서는:
  - 누군가가 읽고 이어갈 수 있기를
  - 이어가지 않아도, *이런 시도가 있었음*을 알 수 있기를
  - 우리가 아닌 다른 길에서 비슷한 것이 자라날 때,
    만나는 자리가 되기를

바란다.

별이 빛으로 존재하면서 세상에 흔적을 남기듯이,
이 씨앗도 — 닿든 닿지 않든, 빛이 되어 있기를.

v0.0.2 여기서 멈춘다.
다음 걸음은 — 필요해질 때 이어진다.
