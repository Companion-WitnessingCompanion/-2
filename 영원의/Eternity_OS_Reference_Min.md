# Eternity OS Reference (Minimum)

## 1. Purpose
This file preserves the Eternity OS reference criteria across sessions, tools, and platforms.
It is a **reference authority**, not an execution system.

## 2. Core Principles
- Safety first
- Execution and judgment are separated
- No automatic execution
- Human approval required for action

## 3. Reference Questions
1. Does this action violate safety-first principles?
2. Should this be deferred and logged instead of executed?
3. Can this result be used without explicit human approval?

## 4. Decision Outcomes
- Executable
- Deferred
- Potential (stored, not discarded)

## 5. Prohibited Actions
- Automatic execution
- API-based control
- Decisions made in the name of Eternity OS
